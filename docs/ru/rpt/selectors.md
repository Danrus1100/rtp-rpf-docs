# Система селекторов

Селекторы - это json файлы, которые могут выбирать и применять объекты. На данный момент их существет 2 типа:
 - Текстурные свапперы (`rpt/swappers`)
 - Трансформации от первого лица (`rpt/first_person`)

## Текстурные свапперы

Текстурные свапперы (далее просто свапперы) позволяют подменять некоторые текстуры в игре в зависмости от связанным с ним предметом.

На момент написания свапперы могут менять 3 вида текстур:
 - Броня игрока (`textures\entity\equipment\humanoid`)
 - Стекло подзорной трубы
 - Текстуру наложения (в ваниле есть только Вырезаная Тыква)


Файлы свапперов должны находится в `assets\namespace\rpt\swappers`. от неё нужно положить **json** файл сваппера в том же пути, что и текстура, которую вы хотите заменить (например, для алмазной внешней брони это будет  от корня набора `assets\minecraft\rpt\swappers\textures\entity\equipment\humanoid\diamond.json`)

Объект Сваппера - это путь до текстуры (например `namespace:full/path/to/texture.png`)

::: warning Обратите внимание
Хотя и можно чисто технически указать `fallback` для свапперов (об этом в ["Синтаксис"](#синтаксис)), но они будут игнорироваться, чтобы другие наборы тоже смогли работать. Пожалуйста, не старайтесь их использовать!
:::

## Трансформации от первого лица

Трансформации от первого лица позволяют переопределять то, как будут распологаться предметы на вашем экране, когда вы смотрите на них от первого лица. Преминимо для любого предмета

Файлы транцформаций находятся `assets\namespace\rpt\first_person`. в неё нужно положить **json** файл в том же пространстве имен и название модели предмета из компонента предмета, который вы хотите изменить (например, для алмаза это будет `assets\minecraft\rpt\first_person\diamond.json`)

Объект трасформации - это json объект с таким синтаксисом:

```json
{
    "main": {
        "rotation": [ // Вращение
            "1+1",
            0,
            2
        ],
        "translation": [ // Положение
            // То же самое, что и в rotation
        ],
        "scale": [ // Размер
            // То же самое, что и в rotation
        ]
    },
    "offhand": {
        // То же самое, что и в "main"
    }
}
```

Значения поддерживают [математические выражения](expressions.md), а так же здесь можно использовать 2 дополнительные переменные игры:
 - `drawProgress`: Прогресс доставания предмета от 0 до 1
 - `rightHand`: если предмет в правой руке - то 1, иначе 0

::: warning Обратите внимание
Данные трасформации не будут адаптироваться к тому, в какой руке будет держаться предмет. Если вы хотите, чтобы предмет отображался в нужной руке, вам нужно прописать оба типа трасформации для каждой руки вручную
:::

## Синтаксис

Синтаксис селекторов напоминает модели предметов, но имеют другие типыЖ 

 - `rpt:apply` применяет объект из `value` без проверок:

```json
{
    "type": "rpt:apply",
    "value": ...
}
```
 - `rpt:component` проверяет компонент связанного с тектурой предмета (идея похожа на модель предмета `select`):

```json    
{
    "type": "rpt:component",
    "component": "custom_name",
    "cases": [
        {
            "when": "hi",
            "child": {...}
        }
    ],
    "fallback": {...}
}
```
Если в `fallback` ничего не указывать, то будет использован пустой сваппер

 - `rpt:empty` - пустой сваппер 

 ```json
 {
    "type": "rpt:empty"
 }
 ```

 - `rpt:expression` - выражения. Те же переменные что и в разделе [Математические выражения](expressions.md#игровые-переменные)

 ```json
 {
    "type": "rpt:expression",
    "value": "1 + 2",
    "cases": [
        {
            "when": 2,
            "child": {...}
        },
        {
            "when": [ "<4" , ">=3" ],
            "all": true,
            "child": {...}
        }
    ],
    "fallback": {...}
 }
 ```

  - `rpt:composite` - применить несколько свапперов одновременно

  ```json
  {
    "type": "rpt:composite",
    "children": [
        {...},
        {...}
    ]
  }
  ```

 Обратите внимание, что этот тип селектора может работать неккоректно в некоторых случаях: например, для сваппера брони будет отрисованна ТОЛЬКО либо первая либо последнияя текстура. Эту функцию стоит использовать для текстур наложения (в том числе и подзорная труба), потому что это изначально разрабатывалась для них
