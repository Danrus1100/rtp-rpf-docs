# Работа с форматом BlockBench

## Загрузка .bbmodel

Все модели формата BlockBench должны храниться в `assets/namecpace/rpt/bb/models/`. Все модели в этой папке будут загружены в память игры.

Чтобы их использовать, вам нужно использовать special модель `rpt:bbmodel`:

```json
{
    "oversized_in_gui": true,
    "model": {
        "type": "minecraft:special",
        "base": "item/...",
        "model": {
            "type": "rpt:bbmodel",
            "model": "model_name"
        }
    }
}
```

## Анимации и Конечные автоматы (FSM)

::: warning Дисклеймер
На данный момент скорость отрисовки bbmodel'ей не очень оптимизирована! даже 3-5 моделй могут "уронить" ваш FPS на 10-40%, так что используйте их очень осторожно. Я уже работаю над тем чтобы это исправить. Так же если у вас есть возомжность, то пожалуйста, не используйте анимации, если то что вы хотите можно сделать и без них, потому что доп. оптимизация выключается, если вы её используйте

Чтобы воспроизводить анимации и переключать их в зависимости от каких либо событий, создана система FSM. Её суть заключается в том, что вам нужно указать состояния, их анимации и то, куда, когда и куда будут переходить состояния.

Файлы определения FSM должны находится в `assets/namecpace/rpt/bb/fsm/`, и иметь такое же название, что и **файл BlockBench** модели

```json
{
    "initial_state": "other_state",
    "states": {
        "my_state": {
            "animation": "cool_animation",
            "transitions": [
                {
                    "to": "other_state",
                    "trigger": "animation_finished" // Так же есть draw (доставание предмета), attack, use
                }
            ]
        },
        "other_state": {
            "animation": "less_coll_animation",
            "transitions": [ // Вы можете оставить это пустым, если хотите чтобы играла одна анимация
                {
                    "to": "my_state",
                    "trigger": "attack"
                }
            ]
        }
    }
}
```

помимо простых триггеров, что указаны выше, есть ещё некоторые более гибкие:
 - `conditional` - тригерит переход, если данное Матемтическое выражение выполняет условие. Сравнивает выражение с 0: 
 ```json
 "trigger": {
    "type": "conditional",
    "condition": "<1" // Триггер сработает, потому что 0 < 1
 }
 ```
 - `keypress` - нажатие на клавишу. О кодах клавиш вы можете узнать в документации [GLFW (англ)](https://www.glfw.org/docs/3.3/group__keys.html):
 ```json
 "trigger": {
    "type": "keypress",
    "key": 32 // Пробел
 }
 ```
